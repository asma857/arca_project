# Politique d’Audit Interne 

## 1. Objet
La présente politique définit le cadre, les responsabilités, les obligations et les processus liés à l’audit interne au sein de l’entreprise.  
Elle vise à :
- Identifier les risques
- Évaluer les contrôles internes
- Assurer la conformité légale
- Améliorer l’efficacité opérationnelle
- Renforcer la gouvernance globale de l’organisation

L’audit interne est un instrument essentiel pour assurer la transparence et la fiabilité des opérations.

## 2. Portée
Cette politique s’applique :
- À tous les départements (IT, RH, Finance, Production, Sécurité…)
- Aux opérations internes et externes susceptibles d’affecter les activités
- Aux systèmes d’information, infrastructures critiques et processus métier
- À tous les employés, prestataires et partenaires travaillant pour l’entreprise

Elle couvre :
- Audits financiers
- Audits opérationnels
- Audits de sécurité informatique
- Audits de conformité légale
- Audits RH
- Audits des fournisseurs

## 3. Responsabilités Générales
### L’équipe Audit Interne doit :
- Définir un programme annuel d’audit
- Identifier les risques prioritaires
- Réaliser les missions d’audit
- Analyser les preuves collectées
- Rédiger des rapports détaillés
- Suivre la mise en place des correctifs

### Les managers doivent :
- Coopérer avec les auditeurs
- Fournir les documents demandés
- Appliquer les recommandations
- Documenter les actions entreprises

### Les employés doivent :
- Faciliter les audits
- Fournir informations et preuves
- Signaler toute non-conformité constatée

## 4. Indépendance de l’Audit Interne
L’audit interne doit rester :
- Objectif
- Neutre
- Indépendant des départements audités

Les auditeurs ne doivent pas auditer un processus qu’ils supervisent eux-mêmes.

## 5. Programme Annuel d’Audit
Le programme doit être :
- Élaboré chaque année
- Basé sur une analyse des risques
- Validé par la direction générale
- Structuré selon les priorités stratégiques

Il doit inclure :
- Objectifs
- Étendue
- Ressources nécessaires
- Calendrier annuel

## 6. Types d’Audits
### 6.1. Audit Financier
Vérifie :
- Exactitude des états financiers
- Respect des normes comptables
- Absence de fraude ou manipulation

### 6.2. Audit Opérationnel
Évalue :
- Efficacité des processus
- Optimisation des ressources
- Respect des procédures internes

### 6.3. Audit de Sécurité Informatique
Analyse :
- Sécurité des systèmes
- Gestion des accès
- Sauvegardes
- Conformité aux politiques IT

### 6.4. Audit de Conformité
Vérifie :
- Application des lois
- Respect des normes sectorielles
- Conformité RGPD
- Conformité réglementaire interne

### 6.5. Audit RH
Contrôle :
- Gestion du personnel
- Respect du droit du travail
- Suivi des absences et formations

### 6.6. Audit des Fournisseurs
Évalue :
- Fiabilité
- Qualité
- Risques liés à la chaîne d’approvisionnement

## 7. Droit d’Accès des Auditeurs
Les auditeurs internes ont accès à :
- Tous les systèmes
- Tous les documents internes
- Toutes les factures, contrats, emails professionnels
- Toutes les zones physiques (salles serveurs, archives)
- Tous les historiques et journaux d’accès

Aucune information ne doit leur être refusée sauf ordre légal.

## 8. Étapes d’un Audit Interne
1. Préparation  
2. Analyse documentaire  
3. Entrevues avec les responsables  
4. Collecte des preuves  
5. Tests de conformité  
6. Analyse des résultats  
7. Rédaction du rapport  
8. Recommandations  
9. Suivi des actions correctives

## 9. Collecte de Preuves
Les preuves peuvent inclure :
- Captures d’écran
- Logs système
- Emails
- Documents internes
- Extraits comptables
- Vidéos de surveillance
- Résultats de tests techniques

Les preuves doivent être :
- Authenticables
- Traçables
- Stockées dans un espace sécurisé
- Protégées par chiffrement

## 10. Tests de Conformité
Les auditeurs doivent :
- Tester les accès utilisateurs
- Vérifier les journaux d’activité
- Tester les sauvegardes et restaurations
- Vérifier l’application des politiques internes
- Comparer les pratiques réelles aux procédures officiellement documentées

Toute déviation doit être signalée dans le rapport.

## 11. Rapports d’Audit
Chaque rapport doit contenir :
- Résumé exécutif
- Méthodologie utilisée
- Constatations principales
- Risques identifiés
- Non-conformités
- Recommandations
- Délai de mise en œuvre
- Priorité (faible, moyenne, élevée, critique)

Les rapports doivent être :
- Signés par l’auditeur responsable
- Transmis à la direction générale
- Archivé pendant 7 ans minimum

## 12. Risques Identifiés
Les risques peuvent inclure :
- Mauvaise gestion des accès
- Absence de sauvegardes
- Processus non documentés
- Dépendance fournisseur élevée
- Matériel obsolète
- Failles de sécurité majeures
- Absence de formation des employés

Les risques prioritaires doivent être traités immédiatement.

## 13. Plan d’Action Correctif
Les actions correctives doivent :
- Être documentées
- Avoir un responsable désigné
- Avoir une date limite fixée
- Être suivies mensuellement
- Être validées à la fin par l’équipe d’audit interne

Tout retard doit être justifié par écrit.

## 14. Suivi et Vérification
Après un audit :
- L’équipe audit interne doit suivre les correctifs
- Faire des contrôles réguliers
- Vérifier les progrès réalisés
- Organiser des audits de suivi si nécessaire

Toutes les preuves de correction doivent être archivées.

## 15. Confidentialité des Audits
Les informations collectées sont strictement confidentielles :
- Interdiction de partager avec des tiers
- Interdiction de stocker sur des appareils personnels
- Interdiction de transférer en dehors des systèmes internes

Toute fuite est considérée comme incident majeur.

## 16. Formation des Auditeurs
Les auditeurs doivent suivre :
- Formations sur les risques
- Formations techniques (cybersécurité, IT)
- Formations juridiques et réglementaires
- Formations méthodologiques (ISO 19011, COBIT, ITIL)

Les compétences doivent être réévaluées chaque année.

## 17. Conflits d’Intérêts
Les auditeurs ne doivent pas :
- Auditer leur propre manager
- Auditer un département où ils ont travaillé récemment
- Recevoir cadeaux, primes, ou avantages de parties tiers

Tout conflit potentiel doit être déclaré.

## 18. Infractions et Sanctions
Toute obstruction à un audit peut entraîner :
- Avertissement
- Suspension
- Sanction disciplinaire
- Poursuites légales en cas de fraude

Toute falsification de preuve est considérée comme faute grave.

## 19. Audit Externe
Des audits externes peuvent être mandatés pour :
- Vérifier la conformité légale
- Auditer la sécurité informatique
- Évaluer l'efficacité globale du contrôle interne

Les résultats doivent être communiqués à la direction.

## 20. Révision de la Politique
Cette politique doit être révisée :
- Tous les 12 mois
- Après un incident majeur
- Après une modification réglementaire
- Suite à une recommandation d’audit externalisé
