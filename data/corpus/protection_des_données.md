# Politique de Protection des Données

## 1. Objet
La présente politique définit l’ensemble des règles, obligations et mécanismes adoptés par l’entreprise pour assurer la protection, la confidentialité, l'intégrité, la disponibilité et la gouvernance des données personnelles et professionnelles. Elle s’applique à toutes les données collectées, stockées, analysées ou transférées par l’entreprise.

## 2. Portée
Cette politique concerne :
- Les employés permanents
- Les contractuels
- Les consultants externes
- Les prestataires ayant accès aux données
- Les systèmes et infrastructures internes ou cloud

Elle couvre également les traitements automatisés, manuels ou hybrides.

## 3. Principes Fondamentaux
L'entreprise s'engage à respecter les principes suivants :
- Minimisation des données
- Finalité déterminée, explicite et légitime
- Limitation de la conservation
- Sécurité renforcée
- Transparence envers les utilisateurs
- Restriction d’accès strict basé sur le besoin professionnel

## 4. Bases Légales du Traitement
Les traitements de données doivent reposer sur l’une des bases suivantes :
- Consentement explicite
- Obligation légale
- Exécution d’un contrat
- Protection des intérêts vitaux
- Mission d’intérêt public
- Intérêt légitime de l’entreprise

Chaque traitement doit être documenté dans le registre de traitement interne.

## 5. Catégories de Données
Les données traitées par l’entreprise sont classées comme suit :
- Données d'identification (nom, email, N° employé)
- Données financières (relevés, factures)
- Données de localisation
- Données sensibles (santé, biométrie, opinions)
- Données techniques (logs, adresses IP, cookies)
- Données RH (contrats, évaluations)

## 6. Droits des Utilisateurs
Les employés, clients ou utilisateurs disposent des droits suivants :
- Droit d'accès
- Droit de rectification
- Droit à l’effacement
- Droit à la portabilité
- Droit à la limitation
- Droit d’opposition
- Droit de retrait du consentement à tout moment

Toute demande doit être traitée dans un délai de 30 jours maximum.

## 7. Sécurité des Données
- Toutes les données sensibles doivent être chiffrées (AES-256).
- Le chiffrement en transit est obligatoire (TLS 1.2 ou supérieur).
- Les bases de données doivent être protégées par un firewall applicatif.
- Les sauvegardes doivent être stockées dans des environnements sécurisés.
- L’accès aux données doit être journalisé.

## 8. Limitation de la Conservation
Les durées de conservation maximales sont :
- Données personnelles : 2 ans après fin de contrat ou finalité
- Données financières : 10 ans
- Données RH : 5 ans après départ
- Données techniques : 90 jours (logs standards)
- Données de sécurité : 180 jours (logs critiques)

Toute donnée expirée doit être supprimée ou anonymisée.

## 9. Partage des Données
Le partage interne ou externe doit respecter :
- Une validation préalable du DPO
- Un accord contractuel ou légal
- Le respect du principe du moindre privilège
- Un chiffrement obligatoire

Aucun transfert n’est permis vers un pays non conforme sans garanties appropriées.

## 10. Violations de Données
Tout incident impliquant des données personnelles doit être :
- Déclaré au DPO dans l’heure
- Documenté dans le registre d’incidents
- Notifié à l’autorité compétente sous 72 heures (si nécessaire)
- Communiqué aux personnes concernées selon la gravité

## 11. Analyse d’Impact – DPIA
Les traitements présentant un risque élevé doivent faire l’objet d’une analyse d’impact (DPIA), notamment :
- Traitement de données sensibles
- Surveillance systématique
- Traitement à grande échelle
- Utilisation d’IA ou d’automatisation décisionnelle

Le DPIA doit être validé par le DPO et révisé périodiquement.

## 12. Sous-Traitants
Tout prestataire manipulant des données doit :
- Signer un DPA (Data Processing Agreement)
- Respecter les mêmes normes que l’entreprise
- Appliquer des mécanismes de sécurité équivalents
- Notifier toute violation dans un délai de 24 heures

## 13. Registre des Traitements
Le registre doit inclure :
- Nom du traitement
- Finalité
- Type de données
- Catégories de personnes
- Durées de conservation
- Sous-traitants impliqués
- Mesures de sécurité

## 14. Audit et Contrôles
- Un audit interne doit être mené chaque année.
- Les accès aux bases de données doivent être revus trimestriellement.
- Les journaux doivent être consultés régulièrement.

## 15. Formation
Tous les employés doivent suivre :
- Une formation initiale RGPD
- Une formation annuelle de rappel
- Un module obligatoire sur la confidentialité des données

## 16. Sanctions
Toute violation de cette politique peut entraîner :
- Avertissement
- Suspension d’accès
- Sanction disciplinaire
- Poursuite interne ou externe selon la gravité

## 17. Révision
Cette politique doit être révisée au minimum tous les 12 mois.
